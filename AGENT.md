# AGENT Guidance for `ads_coding_assessment`

This assessment is designed to evaluate skills and proficiency in the following areas:

## 1. Pharmaverse Ecosystem
- Work with open-source packages designed for clinical trial data standards.
- Focus on SDTM (Study Data Tabulation Model) and ADaM (Analysis Data Model).
- Use relevant tooling such as `{admiral}` and `{sdtm.oak}` where appropriate.

## 2. Data Manipulation & Derivations
- Apply modern R data tools including `{dplyr}` and `{tidyr}`.
- Implement robust, traceable derivations using Pharmaverse-aligned approaches.
- Use packages such as `{gtsummary}` for structured analysis outputs.

## 3. Clinical Reporting
- Generate clear and submission-ready Tables, Listings, and Graphs (TLGs).
- Emphasize outputs that align with regulatory expectations.

## 4. Python for Data Science
- Demonstrate practical Python usage for data science tasks where useful.
- Show ability to integrate Python-based analysis with clinical programming workflows.

## 5. Clean Code Practices
- Write reproducible, efficient, and well-documented R code.
- Prefer readable, modular solutions that are easy to review and maintain.

## 6. Problem-Solving & Adaptability
- Demonstrate ability to quickly learn new tools and apply them effectively.
- Provide thoughtful, complete solutions to unfamiliar challenges.

## 7. Efficient Use of AI
- Leverage modern AI tools to accelerate learning and implementation.
- Use AI responsibly to improve speed without sacrificing quality or correctness.

---

## Most Important Evaluation Priorities

1. **Code Quality**  
   Clarity, efficiency, and adherence to R best practices.

2. **Correctness**  
   Whether the code produces the expected output.

3. **Documentation**  
   Whether functions and code are well-documented.

4. **Problem-Solving**  
   Creativity, rigor, and thoroughness of the approach.

5. **Communication**  
   Ability to clearly explain decisions and implementation approach.
